# 시맨틱 태그(Semantic): 태그 이름만 봐도 의미를 알 수 있다. - 의미의, 의미론적인

: paragraph(단락) : anchor(닻) # 시맨틱 태그 사용 목적 : '웹 문서의 구조'를 만드는 주요 태그에서 사용 - HTML 소스 코드만 보고도 어느 부분이 제목, 메뉴, 본문 내용인지 쉽게 파악 가능 - 문서 구조가 정확히 나눠지기 때문에 PC나 모바일의 웹 브라우저와 여러 스마트 기기의 다양한 화면에서 웹 문서를 표현하기 용이 - 인터넷에서 웹 사이트를 검색할 때 필요한 내용 검색에 용이 - 접근성 향상, SEO 최적화, 코드의 가독성 및 유지보수 용이성 # 웹 문서 구조를 만드는 주요 시맨틱 태그의 종류
header 태그
- 헤더 영역 - 주로 맨 위쪽이나 왼쪽에 위치 - 검색 창이나 사이트 메뉴를 삽입 - 태그와 혼동X (웹 문서의 정보를 담는 태그)
nav 태그
- 네비게이션 영역 - 같은 웹 문서 안에서 다른 위치로 연결 OR 다른 웹 문서로 연결하는 링크를 포함
main 태그
- 메인 콘텐츠 영역 - 메뉴, 사이드 바, 로고처럼 페이지마다 똑같이 들어간 정보 X 웹 문서마다 다르게 보여지는 내용으로 구성 - 웹 문서에서 한 번만 사용!
article 태그
- 웹에서 실제로 보여주고 싶은 내용을 삽입하는 영역 - 웹 문서에서 여러 개 사용 가능 - article 태그 안에 section 태그 사용 - 블로그의 포스트나 뉴스 사이트의 기사처럼 독립된 웹 콘텐츠 항목과 같은 내용 삽입
section 태그
- 콘텐츠 영역 - 몇 개의 콘텐츠를 묶는 용도로 사용 - div 태그: 단순히 원하는 내용들을 묶는 용도로 사용
aside 태그
- 사이드바 영역 - 본문 내용 외에 왼쪽, 오른쪽 혹은 아래쪽에 위치 - 웹 사이트의 필수 요소 X
footer 태그
- 푸터 영역 - 웹 문서 맨 아래 위치 - 사이트 제작 정보, 저작권 정보, 연락처 등을 입력
div 태그
- division: 구분, 분할 -
-
: id, class 속성을 사용하여 해당 영역에 의미를 붙여 사용 : 문서 구조(영역 구별)를 지정
