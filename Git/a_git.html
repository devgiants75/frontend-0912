1. Git 정의 - 분산 버전 관리 시스템(DVCS) : 소스 코드의 변경 내역을 여러 개발자와 효과적으로 공유 관리 2. Git 특징 - 빠른 성능 - 브랜칭과 병합 지원 - 데이터 무결성 - 오프라인 작업 지원 2. GitHub 정의 - Git을 사용하는 프로젝트를 위한 웹 기반 호스팅 서비스 - 코드 관리, 협업, 코드 리뷰, 프로젝트 관리 등의 다양한 기능 제공 // Git 사용 시 필요한 기본 용어 및 개념 // Repository(Repo) : 프로젝트 데이터, 소스코드, 문서 등이 저장되는 장소(디렉터리) 1. working directory : 실제 파일이 위치하는 곳 : git add 명령을 사용하여 스테이징 영역으로 옮김 2. staging area : 커밋을 준비하는 임시 영역(변경사항을 임시로 저장하는 영역) : git commit 명령을 사용하여 리포지토리에 영구히 저장 3. (local) repository : 프로젝트의 모든 버전과 히스토리가 저장되는 곳 : git push 명령을 사용하여 원격 리포지토리에 업로드
