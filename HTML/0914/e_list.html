Lists

Unordered List(순서가 없는 리스트)

- 항목의 순서가 중요하지 않을 때 사용 - unordered list

    - 리스트 종류 안에는 리스트 항목을
  • 태그로 감싸서 사용 li: list item

    • milk
    • eggs
    • bread
    • snack
    - 불릿(bullet): 순서가 없는 목록의 경우 항목 앞에 작은 원이나 사각형을 붙여서 구분

    ordered list(순서가 있는 목록)

    - 각 항목을 순서대로 나열할 때 사용 - ordered list:

      - 리스트 종류 안에는 리스트 항목을
    1. 태그로 감싸서 사용

      1. nc VS 삼성 (영문 - 소문자)
      2. 롯데 VS 두산 (영문 - 대문자)
      3. LG VS KT (로마 숫자 소문자)
      4. KIA VS SSG (로마 숫자 대문자)
      5. 한화 VS 키움 (숫자 - 기본값)
      - 순서가 있는 목록은 기본적으로 숫자 1, 2, 3...으로 번호가 붙는다.
      - type 속성을 사용하여 목록의 순서를 나타내는 표현을 변경할 수 있다. - start 속성을 사용하여 시작 번호를 지정 가능

      definition List(목록 설명)

      - 이름(name)과 값(value)의 형태로 된 목록
      - 웹 문서에서 목록 형식의 콘텐츠를 삽입할 때 사용

      이름(단어명) 부분 지정: definition table
      값(설명) 부분 지정: definition data
      한 쌍의 dt태그와 dd태그를 입력 - dt 태그 하나에 dd태그를 여러 개 사용 가능
      선물용 사과 3kg
      소과 13 ~ 16과
      중과 10 ~ 12과
      선물용 사과 5kg
      중과 15 ~ 19과
